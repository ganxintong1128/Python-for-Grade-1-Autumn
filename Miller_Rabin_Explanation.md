# Miller-Rabin 素数测试算法详解

## 算法概述

Miller-Rabin 是一种概率性素数测试算法，特别适用于测试大数的素性。它基于费马小定理和二次探测定理，具有很高的准确性和效率。

## 数学理论基础

### 1. 费马小定理 (Fermat's Little Theorem)
对于素数 p 和与 p 互质的整数 a：
```
a^(p-1) ≡ 1 (mod p)
```

### 2. 二次探测定理
如果 p 是素数，且 x² ≡ 1 (mod p)，那么 x ≡ ±1 (mod p)

### 3. 算法核心思想
对于奇数 n > 2，可以将 n-1 写成：
```
n-1 = u × 2^t
```
其中 u 是奇数，t ≥ 1

## 代码详解

```python
def mod_pow(base, exp, mod):
    """快速模幂运算 - 计算 base^exp mod mod"""
    result = 1
    base = base % mod
    while exp > 0:
        if exp % 2 == 1:
            result = (result * base) % mod
        base = (base * base) % mod
        exp = exp // 2
    return result
```

**快速模幂运算说明：**
- 使用二进制快速幂算法
- 时间复杂度：O(log exp)
- 避免计算超大数，通过取模保持数值在合理范围

### Miller-Rabin 测试函数

```python
def is_prime(x):
    if x == 2 or x == 3:
        return True
    elif x % 2 == 0:
        return False
    
    # 将 x-1 分解为 u × 2^t 的形式
    u = x - 1
    t = 0
    while u % 2 == 0:
        u //= 2
        t += 1
    
    test_bases = [2, 3]
    for a in test_bases:
        if a >= x:
            continue
        
        # 计算 a^u mod x
        y = mod_pow(a, u, x)
        
        # 检查必要条件1：如果 a^u ≡ 1 或 -1 (mod x)，通过测试
        if y == 1 or y == x - 1:
            continue
        
        # 检查必要条件2：连续平方 t-1 次
        for _ in range(t - 1):
            y = mod_pow(y, 2, x)
            if y == x - 1:  # 找到 -1，通过测试
                break
        else:
            # 如果循环正常结束（没有找到 -1），则不是素数
            return False
    
    return True
```

## 算法步骤详解

### 步骤1：预处理
- 处理特殊情况：2, 3 是素数，偶数不是素数
- 将 n-1 分解为 u × 2^t 形式

### 步骤2：选择测试基数
- 对于不同范围的数，选择不同的测试基数
- 代码中使用 [2, 3] 作为测试基数
- 更大的数需要更多基数以确保准确性

### 步骤3：执行Miller-Rabin测试
对每个测试基数 a：

1. **计算 a^u mod n**
2. **检查条件1**：如果 a^u ≡ 1 (mod n) 或 a^u ≡ -1 (mod n)，通过测试
3. **检查条件2**：连续平方 t-1 次，如果某次得到 -1，通过测试
4. **如果所有测试都没有找到 -1，则判定为合数**

## 为什么这样工作？

### 理论依据
1. **如果 n 是素数**：
   - 根据费马小定理：a^(n-1) ≡ 1 (mod n)
   - 由于 n-1 = u × 2^t，所以 a^(u×2^t) ≡ 1 (mod n)
   - 在连续平方过程中，必然会遇到 ±1

2. **如果 n 是合数**：
   - 很可能违反上述条件之一
   - 通过多个基数测试，错误率极低

### 错误概率
- 单次测试的错误概率 ≤ 1/4
- k 次独立测试的错误概率 ≤ (1/4)^k
- 使用 [2, 3] 两个基数，错误概率 ≤ 1/16

## 算法优势

1. **高效性**：时间复杂度 O(k log³ n)，其中 k 是测试轮数
2. **准确性**：选择合适的基数可以达到确定性结果
3. **适用性**：特别适合大数素性测试
4. **实用性**：广泛应用于密码学和数论
